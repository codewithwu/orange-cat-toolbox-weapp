# 这是个微信小程序项目

## 技术栈
- 原生微信小程序开发（不是 uni-app 也不是 Taro）
- 使用 npm 包管理依赖

## 你一定要遵守的规矩
- **路径规范**：所有页面必须在 `pages/` 目录下，组件在 `components/`。
- **样式注意**：尺寸单位统一用 `rpx`，不要用 `px`。
- **API 请求**：统一封装在 `utils/request.js` 里，不要在页面里直接写 `wx.request`。
- **代码风格**：变量用小驼峰 `myData`，方法名用 `handleClick`。

## 常用命令
- 启动构建（监听 npm 包变化）：`npm run watch`
- 在微信开发者工具里点“编译”，别在终端里瞎喊。

## 雷区（千万别碰）
- **不要修改 `project.config.json`**，除非我明确让你改。
- 别轻易升级基础库版本。

## 安全规范
- 将代码推送到 git 仓库前，必须检查是否有敏感信息（如 appid、secret、密钥等），如有发现，及时提醒用户处理后再推送。
